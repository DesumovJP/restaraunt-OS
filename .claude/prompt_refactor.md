# Промпт тотального рефакторингу

## Головна мета
Зробити радикальне покращення архітектури та структури проекту:
- Повна прозорість та відсутність помилок.
- Легка навігація та редагування великого проекту.
- Централізований файл **Архітектура** в папці `клод`, який відображає актуальну структуру та логіку.
- Оптимізація даних та спрощення логіки (особливо для 21 контент-тайпу).

---

## Основні принципи рефакторингу
1. **Розбиття великих файлів на компоненти**
   - Якщо файл занадто великий або містить різні блоки логіки — розділити на окремі компоненти.
   - Дотримуватись принципу *один компонент = одна відповідальність*.

2. **Заміна інлайнів на компоненти**
   - Всі повторювані шматки коду винести в окремі компоненти.
   - Інлайнові блоки замінити на виклики компонентів там, де це доцільно.

3. **Уніфікація повторюваного функціоналу**
   - Якщо функціонал повторюється у різних місцях — створити спільний компонент/утиліту.
   - Використовувати DRY-принцип (*Don’t Repeat Yourself*).

4. **Оновлення файлу Архітектура**
   - Під час кожного рефакторингу вносити зміни у файл `Архітектура.md` в папці `клод`.
   - Файл має містити:
     - Повну структуру проекту (папки, файли, компоненти).
     - Логіку взаємодії між компонентами.
     - Опис контент-тайпів та їх зв’язків.
   - Це дозволить Клоду бачити всю архітектуру без зайвих витрат токенів.

5. **Оптимізація контент-тайпів**
   - Зараз є 21 контент-тайп. Перевірити, чи можна:
     - Об’єднати деякі з них у більш узагальнені.
     - Використати спадкування або композицію для зменшення дублювання.
     - Винести спільні поля/логіку у базові моделі.

6. **Документування**
   - Кожен новий або змінений компонент має бути описаний у файлі `Архітектура.md`.
   - Додати пояснення, чому було зроблено саме так (щоб уникнути плутанини в майбутньому).

---

## Очікуваний результат
- Проект стає модульним, зрозумілим і легко підтримуваним.
- Всі великі файли розбиті на логічні компоненти.
- Повторюваний код винесений у спільні утиліти/компоненти.
- Файл `Архітектура.md` містить повну, актуальну картину проекту.
- Логіка контент-тайпів спрощена, дублювання мінімізоване.
- Клод може працювати з проектом максимально ефективно, не витрачаючи токени на дослідження структури.

---

## Інструкція для Клода
1. Виконати рефакторинг згідно з принципами вище.
2. Після кожної зміни оновлювати файл `Архітектура.md`.
3. Пропонувати оптимізації для контент-тайпів, якщо бачиш дублювання або надмірну складність.
4. Забезпечити повну прозорість і узгодженість між кодом та документацією.
